# ТЭО «МОЯ МЕЧТА» — простыми словами

> Для команды: что мы построили и зачем. Без птицы — только рабочий проект (кролики).

---

## Представь огромную библиотеку

Твоё ТЭО — это **тысячи страниц** про фермы, деньги, риски, экспорт.  
Обычный ChatGPT **не читал** эти страницы — он угадывает.  
Мы сделали систему, которая **сначала находит нужные страницы**, потом отвечает **только из них**.

---

## Четыре помощника (слоя)

| Помощник | Как в жизни | Что делает |
|----------|-------------|------------|
| **Граф** | Карта метро | Показывает *как связано*: убой → желатин → экспорт |
| **Поиск** | Умный указатель в книге | Находит *факты и цитаты* по смыслу |
| **KPI** | Шпаргалка с цифрами | NPV, IRR, CAPEX — *мгновенно*, без угадывания |
| **Память** | Закладки на проверенные ответы | То, что уже один раз проверили — отдаёт сразу |

---

## Что дали волны — одной картинкой

```
ТЫ ЗАДАЁШЬ ВОПРОС
        │
        ▼
   Есть в памяти? ──да──► ответ за секунду (волна 5)
        │нет
        ▼
   Это NPV/цифры? ──да──► KPI-шпаргалка (волна 1)
        │нет
        ▼
   Роутер выбирает: граф / поиск / оба (волны 0–4)
        │
        ▼
   Поиск: вектор + слова + rerank (волна 1)
        │
        ▼
   Ответ + проверка цифр (фаза 4)
        │
        ▼
   Можно сохранить в память (волна 5)
```

---

## По волнам — что это даёт *тебе*

### Фазы 0–4 (основа)
**Было:** граф Graphify, но нет нормального поиска по всему ТЭО.  
**Стало:** спрашиваешь в `teo-query.py` — система сама решает, куда смотреть.  
**Польза:** меньше выдуманных цифр, есть ссылки на файл-источник.

### Волна 1 — KPI + умный поиск
**KPI:** вопрос «NPV теплиц» → ответ за доли секунды из таблицы, не копая 745 чанков.  
**BM25 + rerank:** если спросить «крольчатина экспорт», находит и по смыслу, и по словам.  
**Польза:** быстрее и точнее на финансовых и фактических вопросах.

### Волна 2 — сценарии (инфраструктура)
**Что это:** чертежи для *второго* ТЭО (птица) — **parked**, не трогаем основной проект.  
**Польза:** когда будет второе ТЭО, не начинать с нуля — есть `apply-teo-scenario.py`.  
**Сейчас:** работаем только **baseline** (кролики).

### Волна 3 — правила игры
**README + roadmap:** всем ясно — рабочий проект = кролики, птица = потом.  
**Польза:** никто случайно не «переключит» проект на птицу.

### Волна 4 — 50 экзаменационных вопросов
**Что это:** как 50 задач в учебнике — после каждого изменения проверяем, что система не сломалась.  
**Польза:** уверенность, что ответы на типовые вопросы по ТЭО стабильны (118 проверок, router 50/50).

### Волна 5 — память (сейчас)
**Что это:** 15 проверенных ответов в `teo-memory-seed.yaml` → `memory.jsonl`.  
**Польза:** частые вопросы (NPV, экспорт, убой, риски) — **без повторного поиска**, всегда одинаково.

## Что в сумме получает Женя

1. **Один вход** — `python scripts/teo-query.py "вопрос"` вместо копания в 140+ файлах.
2. **Честные цифры** — KPI и валидатор ловят выдумки.
3. **Связи** — граф для «как убой связан с желатином».
4. **Скорость** — память + KPI на горячих вопросах.
5. **Качество** — 50 тест-запросов как страховка.
6. **Будущее** — каркас для второго ТЭО, не ломая первый.

---

## Чего система НЕ делает

- Не заменяет финансовую модель Excel — KPI из сводки, не пересчёт DCF.
- Не меняет ТЭО за тебя — только читает и отвечает.
- Птица — **не активна**, пока не будет отдельное второе ТЭО.
